# forge 사용법 영상 내레이션은 로컬 대신 클라우드 Microsoft Edge TTS로 전환 (ADR-0001 로컬 전용 원칙의 명시적 예외)

`forge-usage-workflows-video`/`voice-clone-redub-pipeline`(사용법 설명 영상 내레이션)에서 GPT-SoVITS 클론(ADR-0001) → Qwen3-TTS 프리셋(ADR-0002) → CosyVoice2 클론(ADR-0003)까지 세 번 로컬 백엔드를 바꿨음에도 항목별 억양·속도 일관성 문제가 반복됐다. 이번엔 **Microsoft Edge TTS(클라우드 뉴럴 TTS)**로 전환하기로 했다.

ADR-0001의 핵심 우려는 "사용자 본인 목소리(클론 참조 샘플)가 기계 밖으로 나가는 것"이었다. Edge TTS는 음성 클론이 아니라 프리셋 뉴럴 보이스라서 사용자 목소리 자체는 전송되지 않고 **대본 텍스트만** 클라우드로 전송된다 — ADR-0001이 막으려던 프라이버시 리스크와는 성격이 다르다. ADR-0001은 "클라우드 전환은 사람이 명시적으로 결정할 때만 한다"고 명시했고, 이번이 그 명시적 결정이다.

**범위**: 이 결정은 사용법 설명 영상의 **비-클론(프리셋) 내레이션**에만 적용된다. `voice-clone-redub-intro` 등 **사용자 본인 목소리 재현이 목적인 음성 클론 작업에는 ADR-0001의 로컬 전용 제약이 그대로 유효**하다 — 이 ADR은 ADR-0001을 대체(supersede)하지 않는다.

## Considered Options
- **Qwen3-TTS 프리셋 파라미터 재조정(로컬 유지)** — ADR-0003에서 이미 동일 문제로 기각된 이력 반복 위험.
- **CosyVoice2 클론 유지, 속도만 조정(로컬 유지)** — 백엔드 자체 일관성 문제는 해결되지 않음.
- **Microsoft Edge TTS(클라우드, 선택)** — 상용급 안정적 뉴럴 보이스, 무료, rate 파라미터로 길이 조절 용이. 대본 텍스트가 외부로 전송된다는 트레이드오프 수용(대본은 공개 교육 콘텐츠).

## Consequences
- ADR-0001의 로컬 전용 제약은 음성 클론 작업(예: `voice-clone-redub-intro`)에는 변함없이 유효.
- 사용법 설명 영상 내레이션에 한해 클라우드 TTS 사용 선례가 생김 — 이후 유사 비-클론 내레이션 작업에서 재검토 가능.
