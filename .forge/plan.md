# forge 사용법·유형별 워크플로우 PPTX → 음성 영상 제작

## 목표 / 비목표
- **목표**: 이미 완성된 `forge-사용법-워크플로우.pptx`(32장)를 입력으로 받아, 한국어 내레이션이 포함된 영상 `forge-사용법-워크플로우.mp4`(1920×1080 H.264)를 제작한다. 내레이션은 슬라이드 발표자 노트를 시청자용 대본으로 재작성한 뒤 edge-tts(ko-KR-SunHiNeural)로 합성하고, 각 슬라이드 화면·음성·자막을 동기화하며, 크로스페이드 전환과 잔잔한 CC0 배경음악 믹싱을 더한다. 분량 약 15~20분.
- **비목표**:
  - PPTX 슬라이드 내용·디자인 자체 수정 (PPTX는 입력 고정, 영상만 산출).
  - PPTX 발표자 노트 원문 수정 (재작성 대본은 별도 산출물이며 노트를 건드리지 않음).
  - 사람 보이스오버 녹음 (TTS 음성만).
  - 다국어 자막·영문 번역 (한국어 음성+자막만).
  - 상용 편집 소프트웨어(프리미어 등) 사용 (ffmpeg 파이프라인만).
  - 영상 썸네일·유튜브 업로드 등 배포 작업.
  - `.forge/CONTEXT.md` 용어집이나 ADR 새로 작성 (미디어 생성 작업, 도메인 용어·아키텍처 결정 아님).

- **스타일/파이프라인 참고**: `.forge/retro/2026-06-28-forge-usage-workflows-pptx.md` — 검증된 렌더 파이프라인(PowerPoint → PDF → PNG, 앱 재기동 후 /tmp 복사본 오픈), 검증 전략(자동 + 육안 교차), 외부 다운로드/설치 게이트 회피 교훈, "내용 적합성 리뷰 지점" 교훈.
- **완료 정의**: `forge-사용법-워크플로우.mp4`가 생성되어 있고, 1920×1080 H.264 MP4이며, 32슬라이드 분량(약 15~20분)으로 각 슬라이드에 (a) 재작성된 한국어 내레이션(edge-tts SunHi), (b) 해당 슬라이드 화면, (c) 하단 자막 burn-in이 음성에 동기화되어 들어 있고, 크로스페이드 전환과 잔잔한 CC0 배경음악 믹싱이 포함되어 있으며, 재생·동기·가독성 검증을 통과한다.

## 합의된 설계
- **대본**: 발표자 노트(발표자 코칭 메모 — "강조하세요/가정합니다/언급하세요" 등 지시어 포함)를 시청자용 내레이션으로 재작성. 발표자 지시어·메타 문구 제거, 자연스러운 구어체(시청자에게 말하듯). 슬라이드별 1개 대본, 32개.
- **음성**: edge-tts `ko-KR-SunHiNeural`로 슬라이드별 mp3 생성(샘플 비교로 확정: say Yuna 대비 자연스러움 우세). 이미 설치됨(외부 설치 게이트 無).
- **화면**: 직전 회고 검증 파이프라인 재사용 — PowerPoint(재기동 후 /tmp 복사본 오픈) → PDF(`POSIX file`) → PNG 1920×1080, 슬라이드별 32장.
- **동기화**: 슬라이드 N의 음성 길이 = 슬라이드 N 이미지 노출 시간. 슬라이드별 음성·이미지 쌍을 ffmpeg로 연결.
- **연출(풀)**:
  - 전환: 슬라이드 간 크로스페이드(ffmpeg `xfade`).
  - 자막: 재작성 대본으로 SRT 생성 → `subtitles` 필터로 burn-in. 하단 중앙, 반투명 어두운 박스, 흰색 한국어 시스템 폰트(`force_style`).
  - 배경음악: CC0 소스(Pixabay Music 등)에서 잔잔한 미니멀/앰비언트 곡 다운로드 → 음성 아래 -20~-25dB로 낮춰 루프·페이드 믹싱(`volume`+`amix`, `-stream_loop`).
- **산출물**: `/Users/gyuha/workspace/forge-learn/forge-사용법-워크플로우.mp4`
- **검증**: 직전 회고 교훈 적용 — 자동 검증(ffprobe 길이/해상도/코덱, 슬라이드 수, 오디오 트랙 존재) + 육안 검수(재생, 슬라이드-음성-자막 동기, 자막 가독성, 배경음 볼륨 적정) 교차. 내용 적합성(대본·자막이 발표 의도와 맞는가)은 육안 리뷰 지점으로 명시.

## 작업 조각
- [ ] S1. **내레이션 대본 재작성** — python-pptx로 32개 발표자 노트 추출 후, 시청자용 내레이션으로 슬라이드별 재작성(발표자 지시어 제거·구어체). 산출: 32개 대본(대본 파일 + 슬라이드 인덱스 매핑). — 완료 기준: 32슬라이드 각각 비어있지 않은 시청자용 대본이 있고, 발표자 코칭 지시어("강조하세요/가정합니다" 등)가 제거되었으며, 재작성 대본이 노트 내용의 의미를 보존한다.
- [ ] S2. **슬라이드별 음성 합성** — edge-tts(ko-KR-SunHiNeural)로 S1 대본 32개를 슬라이드별 mp3로 생성. 각 음성 길이 기록(동기화용). — 완료 기준: 32개 mp3가 생성되었고, 전부 한국어 음성이며, 각 파일의 길이(ffprobe)가 기록되어 있다. (depends: S1)
- [ ] S3. **슬라이드 화면 렌더** — PowerPoint 재기동 후 pptx를 /tmp로 복사해 오픈 → PDF(`POSIX file`) → PNG 1920×1080, 슬라이드별 32장. — 완료 기준: 32장 PNG가 1920×1080 해상도로 생성되었고, 한국어 깨짐·스테일 렌더(옛 문서)가 없음을 확인한다. (S1·S2와 독립, 병렬 가능)
- [ ] S4. **배경음악 확보** — CC0 소스(Pixabay Music 등)에서 잔잔한 미니멀/앰비언트 곡 1개 다운로드, 라이선스(CC0) 확인 기록. 외부 다운로드 게이트·저작권 리스크 발생 시 다른 CC0 소스로 우회하지 말고 중단·보고(회고 교훈 준수). — 완료 기준: 1개 배경음 파일과 CC0 라이선스 출처가 기록되어 있다. (독립)
- [ ] S5. **영상 합성** — ffmpeg로 (a) 슬라이드 이미지×음성 동기 결합, (b) 크로스페이드 전환, (c) 자막 SRT burn-in, (d) 배경음악 믹싱(-20~-25dB, 루프·페이드) → `forge-사용법-워크플로우.mp4`(1920×1080 H.264) 산출. — 완료 기준: mp4가 생성되었고 ffprobe로 1920×1080/H.264/오디오 트랙 존재가 확인되며, 총 길이가 약 15~20분 범위다. (depends: S2, S3, S4)
- [ ] S6. **검증** — 자동 검증(ffprobe: 해상도·코덱·길이·오디오 트랙; 슬라이드-음성 동기 타이밍 계산) + 육안 검수(재생, 슬라이드·음성·자막 동기, 자막 가독성, 배경음 볼륨 적정, 내용 적합성) 교차. 검증 결과 실행 노트에 기록. — 완료 기준: 자동 검증 항목 전부 통과하고, 육안 검수에서 동기·가독성·음량·내용 적합성 이상이 없음이 확인된다. (depends: S5)
