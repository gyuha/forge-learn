# 2026-06-28 — forge 사용법·유형별 워크플로우 안내 PPTX 32장 제작

## 계획 vs 실제
- 계획대로 된 것: 32장(30~35 범위) · 노트 32/32 비어있지 않음 · 미디어 10장 · README 7개 상황(셋업/일상/사소한/무인/재진입/마무리/유지보수) 전부 + 설치 1장 + 신뢰 사다리 L1/L2/L3 + 상태 흐름 2장 + 치트시트 2장 + 출처 · 명령 `/fg-*` 기본 표기(/forge:fg-* 병기는 주석) · 배경이론 제외 원칙 · 이미지 내 명령어/텍스트/상태명/화살표 금지 원칙 — 완료 정의 전 항목 충족.
- 계획과 달랐던 것:
  1. **pptxgenjs → python-pptx 전환**: `npm install pptxgenjs`가 외부 설치 게이트(Code from External)에서 거부됨. 우회 설치 없이 이미 설치된 python-pptx + `notes_slide.notes_text_frame` API로 빌드 방식 전환. 발표자 노트 요구사항은 동일 충족 (Open XML 수동 주입 불필요).
  2. **codex-image 일괄 타임아웃**: 10장을 한 codex exec 호출로 묶자 590초 제한에서 타임아웃. 다른 이미지 생성기로 우회하지 않고(계획 합의 준수) 같은 요구사항의 실행 단위를 1장씩으로 쪼개 재시도로 해결.
  3. **PowerPoint AppleScript PDF 저장 경로**: `save ... in "문자열경로"`가 PowerPoint 컨테이너 내부에 써서 파일이 안 생김 → `in POSIX file` 지정 + 홈 디렉토리 출력으로 해결. 매 렌더 전 PowerPoint quit 후 새 pptx를 /tmp로 복사해 오픈.
  4. **비전 모델 한국어 처리 불안정**: 시각 QA 에이전트 4개 중 1개는 픽셀 자체를 못 봤고, 나머지도 한국어 OCR 난독("산댕쟉" 등)이 심해 비전 보고 신뢰도 낮음 → 소스(build.py/content.py) EMU 단위 끝 위치 계산을 검증 1차 기준으로 채택, 비전은 보조로 강등. 이 기준으로 레이아웃 수정 5건(r_table 행간 gap · r_section 40→34pt · r_scenario 예시 박스 확대 · r_concept 본문 영역 비례화 · 홀수 카드 가운데 정렬)을 잡아냄.
  5. **사후 내용 수정**: slide-31 "흔한 실수" 카드 4(내부 구현 과독)·5(이미지에 명령 박기)이 forge *사용자*의 실수가 아니라 덱 *작성* 메타 조언이라 주제에 안 맞음 → 제거(5→3개, 제목도 수정), r_concept에 n==3 분기 추가. 비전+소스계산 검증이 통과해도 "내용이 주제에 맞는가"는 못 잡음 — 사용자 육안·내용 검수가 빈틈을 잡은 사례.

## 학습
- 다음에 다르게 할 것:
  1. **PPTX 작업은 처음부터 python-pptx로 계획** — `pptxgenjs`(npm) 시도 자체를 스킵. forge-learn 환경에서 외부 npm 패키지 설치 게이트가 pptxgenjs를 매번 막으므로(직전 forge-intro-pptx에 이어 두 번 연속 동일 게이트 hit), python-pptx를 발표자료 빌드 디폴트로固化.
  2. **codex-image는 처음부터 1장씩** — N장을 한 codex exec 호출에 묶으면 590초 타임아웃. 빌드 계획 단계부터 이미지 1장 = 1 실행 단위로 잡을 것 (병렬은 호출 분리로).
  3. **PPTX 검증 = 소스 계산 1차 기준, 비전 보조** — 발표자료 시각 QA에서 비전 모델의 한국어 픽셀/OCR 신뢰도가 낮음이 재확인됨. 레이아웃 오버플로는 빌드 소스의 EMU 단위 끝 위치 계산으로 확정 판정하고 비전은 발견/보조용으로만. 발표자료 작업 표준 검증 전략.
  4. **내용 적합성 리뷰 지점을 빌드 단계에 삽입** — "오버플로 없음·자동 검증 통과"가 "내용이 청중 시나리오에 맞는가"와 무관함. 카탈로그/리스트형 슬라이드(흔한 실수·체크리스트·Q&A 등)는 각 항목이 실제 청중(여기선 forge 사용자) 시나리오에 해당하는지 1차 빌드 직후 *사용자 리뷰 지점*으로 명시.
- **긍정 — 회고 문화 작동 증거**: divergence #3(PowerPoint `POSIX file` + 앱 재기동 후 렌더)은 직전 `forge-intro-pptx` 작업의 run.md 교훈을 그대로 재적용해 한 번에 해결함. 과거 run 노트가 다음 작업 비용을 실제로 줄인 사례. (참고: forge-intro-pptx는 `retro: skipped` 상태라 해당 교훈은 run.md에만 존재 — 일괄 승급 후보.)
