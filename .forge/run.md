# 실행 노트 — 내 목소리 클론으로 사용법 영상 재더빙 + 재사용 파이프라인

실행: 2026-06-29~30 · 방식: **직접 실행**(Dynamic Workflow 미사용 — 순차 의존·로컬 도구·fan-out 이득 없음, 직전 영상 작업 회고 precedent)

## 계획대로 된 것
- S1 로컬 백엔드: uv Python 3.11 venv + torch 2.12(MPS) + GPT-SoVITS v2 + 한국어 g2p(g2pk2/ko_pron/mecab) 정상. 사전학습 모델(BERT 621M·HuBERT 180M·s1 148M·s2 101M) 다운로드.
- S3 32/32 대본 클론 합성(총 19.0분 분량), 누락·이상 없음. `out_audio/NN.wav` + `durations.json`.
- S4/S5 재타이밍 + 재합성: `forge-사용법-워크플로우-내목소리.mp4`(1920×1080 H.264 + AAC + 한국어 soft 자막), 길이 1141.8초.
- 원본 `forge-사용법-워크플로우.mp4` 미변경(90,679,205바이트·6/28 그대로). 새 파일로만 산출 — 계획 준수.
- 재사용성: `tts/run.sh`(러너) + `README.md` + `.gitignore`(무거운 산출물 제외). voice-samples 교체 후 재실행으로 재클론.
- ADR-0001 준수: 전부 로컬, 외부 전송 0, 자동 클라우드 전환 없음.

## 계획과 달랐던 것 / 현장 결정
1. **참조 길이 강제 제약**: 계획은 "≤10초"였고 사용자는 19.9초를 제공. GPT-SoVITS가 참조 3~10초를 **코드로 강제**(`TTS.py:817` 범위 밖 에러) → S2 정규화에서 무음 경계(8.96초)로 트림 필수. 계획의 "필요 시 정규화"가 선택이 아니라 필수였음.
2. **whisper 한국어 오인식**: faster-whisper(small)가 절단 위치는 잘 잡았으나 전사 오류("하네스 란"→"S라는" 등). whisper는 **절단 위치 산출에만** 쓰고 prompt_text는 알려진 원본 전사로 수기 교정.
3. **M5 속도 — 리서치 [알 수 없음] 해소**: 콜드/짧은 텍스트 RTF 2.44였으나 워밍업 후 실제 배치 **RTF ~0.37**(19분 분량을 ~7분에 합성). M5에서 충분히 실용적 — ADR-0001의 "너무 느리면 중단" 미발동.
4. **설치 게이트 — 우려했으나 미차단**: 리서치·이전 회고가 경고한 외부 설치 게이트가 uv/pip·HF·GitHub를 막지 않음. 계획에 없던 의존성 추가: `torchcodec`(torchaudio 2.11 로딩 백엔드), `faster-whisper`(참조 정렬), nltk 리소스(영어 g2p), `fast_langdetect` 모델 캐시 디렉터리. 컴파일 위험 의존성(일본어 `pyopenjtalk`, opencc 소스강제)은 한국어에 불필요해 제외(언어 모듈 지연 로드라 안전).
5. **자막 재타이밍 방식**: 원본 325블록 SRT를 분할·재매핑하는 대신, scripts.json을 구문 단위로 쪼개 새 음성 길이에 **비례 배분**(329블록). 자체 완결적이고, 결과 타이밍이 원본과 거의 일치(블록1: 0→4.811s vs 원본 0→4.806s).
6. **오디오 음량(검수 포인트)**: 최종 믹스 mean -30.3dB / max -9.4dB. amix 기본 정규화(÷입력수)로 전체가 ~6dB 낮아짐 — 다소 조용할 수 있음. **육안/청취 검수에서 판단, 필요 시 라우드니스 부스트 1줄 재믹스**(미실행, 사용자 판단 대기).
